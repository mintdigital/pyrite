Changelog to date

Pyrite is under constant development. Everytime we encounter a new problem - be it Selenium being mental, emulating certain actions a user might do or finding the proces of rining the tests too hard - we try our best to fix it. Here, in a nutshell, is the progress of Pyrite from it's inception as a gem (v0.5.0) to it's current jQuery-enabled state (v0.6.4, or 'The Bad Boy' as I like to call it).

0.5.0

  • Gemify the plugin (morphed from scripts) into something not totally shamefull

0.5.1

  • Have a re-think about how we start the server, use a better stratergy in determining what to `wait_for`
tags missing...

0.5.5

  • The CSS commit - we took the decision that Pyrite should do all it's element locating via CSS. We have a strong front-end team and a pretty handy back-end team too. And no one likes XPath.

0.5.6

  • Add `wait_for :javascript` - This commit allowed us to tell Pyrite which JS frame work we were using (Selenium only supports jQuery or Prototype), which meant we could properly wait for ajax calls to return (no mre `sleep 2`).

0.5.7 & 0.5.8
